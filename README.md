# Product Manager

Учебный backend-проект на Python для управления товарами.

Проект создан для практики базовой серверной логики: CRUD-операций, валидации данных, обработки ошибок, работы с JSON-хранилищем, модульной структуры проекта и основ ООП.

## Возможности

- получение списка товаров;
- поиск товара по `id`;
- создание товара;
- обновление товара;
- удаление товара;
- фильтрация доступных товаров;
- валидация входных данных;
- единый формат ответа для успешных и ошибочных операций;
- сохранение данных в JSON-файл;
- демонстрационный запуск через `main.py`.

## Описание файлов

`app/models.py` — содержит модель товара `Product`.
В этом файле описана структура товара и методы для обновления данных и преобразования объекта в словарь.

`app/services.py` — содержит сервисный слой `ProductService`.
Здесь находится основная логика проекта: получение списка товаров, поиск по `id`, создание, обновление, удаление и фильтрация доступных товаров.

`app/storage.py` — отвечает за работу с JSON-хранилищем.
Файл содержит функции для загрузки товаров из `data/products.json`, сохранения изменений и сброса данных к начальному состоянию для демонстрационного запуска.

`app/validators.py` — содержит функции валидации входных данных.
Проверяется название товара, цена и количество.

`app/responses.py` — содержит единый формат ответа для успешных и ошибочных операций.

`app/main.py` — точка входа в приложение.
В этом файле запускается демонстрационный сценарий работы проекта.

`data/products.json` — файл с данными о товарах.

## Как запустить проект

Клонировать репозиторий:

```bash
git clone https://github.com/texze2012/product-manager-api.git
```

Перейти в папку проекта:

```bash
cd product-manager-api
```

Запустить демонстрационный сценарий:

```bash
python -m app.main
```

На Windows также можно использовать:

```bash
py -m app.main
```

## Пример работы

При запуске проекта выполняется демонстрационный сценарий:

* вывод списка товаров;
* создание нового товара;
* попытка создать товар с некорректной ценой;
* поиск товара по `id`;
* поиск несуществующего товара;
* вывод доступных товаров;
* обновление товара;
* удаление товара;
* вывод итогового списка товаров.

Пример вывода:

```text
=== Список товаров ===
Успех:
[...]

=== Создание товара ===
Успех:
{'id': 4, 'name': 'Keyboard', 'price': 3500, 'quantity': 4, 'is_available': True}

=== Создание товара с некорректной ценой ===
Ошибка:
Цена должна быть больше 0

=== Поиск несуществующего товара ===
Ошибка:
Товар не найден
```

## Формат ответа

В проекте используется единый формат ответа для всех операций.

Успешный ответ:

```python
{
    "success": True,
    "data": {...},
    "error": None
}
```

Ответ с ошибкой:

```python
{
    "success": False,
    "data": None,
    "error": "Текст ошибки"
}
```

Такой подход позволяет одинаково обрабатывать успешные и ошибочные операции.

## Пример товара

```json
{
    "id": 1,
    "name": "TV",
    "price": 30000,
    "quantity": 6,
    "is_available": true
}
```

Поля товара:

* `id` — уникальный идентификатор товара;
* `name` — название товара;
* `price` — цена товара;
* `quantity` — количество товара;
* `is_available` — доступность товара, рассчитывается на основе количества.

---

## Валидация данных

При создании и обновлении товара проверяются входные данные:

* название должно быть строкой и не может быть пустым;
* цена должна быть числом больше 0;
* количество должно быть целым числом и не может быть меньше 0.

Если данные некорректны, сервис возвращает ошибку в едином формате ответа.
